# CI/CD Setup Instructions

## GitHub Secrets Configuration

### Email Notification Setup

1. Go to your GitHub repository
2. Navigate to Settings > Secrets and variables > Actions
3. Click "New repository secret"
4. Add the following secrets:

   - **EMAIL_USERNAME**: Your Gmail address (e.g., yourname@gmail.com)
   - **EMAIL_PASSWORD**: Gmail App Password (not your regular password)
   - **EMAIL_TO**: Recipient email address

#### How to Generate Gmail App Password:
1. Go to your Google Account settings
2. Enable 2-Step Verification
3. Go to Security > 2-Step Verification > App passwords
4. Select "Mail" and "Other (Custom name)"
5. Copy the generated 16-character password
6. Use this as EMAIL_PASSWORD secret

### Slack Notification Setup

1. Create a Slack Incoming Webhook:
   - Go to https://api.slack.com/apps
   - Click "Create New App" > "From scratch"
   - Name your app (e.g., "Test Automation Bot")
   - Select your workspace
   - Click "Incoming Webhooks" from the left menu
   - Toggle "Activate Incoming Webhooks" to ON
   - Click "Add New Webhook to Workspace"
   - Select the channel where notifications should be posted
   - Copy the Webhook URL

2. Add Slack Webhook to GitHub Secrets:
   - Go to Settings > Secrets and variables > Actions
   - Click "New repository secret"
   - Name: **SLACK_WEBHOOK_URL**
   - Value: Paste your Slack Webhook URL

### GitHub Pages Setup (for Allure Reports)

1. Go to Settings > Pages
2. Under "Source", select "GitHub Actions"
3. Save the settings
4. After the first workflow run, reports will be available at:
   https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/

## Local Allure Report Generation

### View Allure Report Locally:
```bash
# Run tests
mvn clean test

# Generate and open report
mvn allure:serve
```

### Generate Static Report:
```bash
mvn allure:report
```
Report will be in: target/site/allure-maven-plugin/

## Testing Notifications

### Test Email:
- Push a commit to trigger the workflow
- Check the recipient email inbox

### Test Slack:
- Push a commit to trigger the workflow
- Check the configured Slack channel

## Troubleshooting

### Email not sending:
- Verify Gmail App Password is correct
- Check if 2-Step Verification is enabled
- Ensure EMAIL_USERNAME is the full email address

### Slack not working:
- Verify Webhook URL is correct
- Check if the Slack app has permission to post in the channel
- Test the webhook URL using curl:
```bash
curl -X POST -H 'Content-type: application/json' --data '{"text":"Test message"}' YOUR_WEBHOOK_URL
```

### Allure report not generating:
- Ensure tests ran successfully
- Check if allure-results directory exists
- Verify Allure is installed in Docker container
